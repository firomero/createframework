Crea tu propio framework, con lo último de los componentes de Symfony2.

Felix Ivan Romero Rodríguez Junio 22, 2014 Patrones, Symfony 3 Comments

De la serie original de Fabien en http://fabien.potencier.org/article/50/create-your-own-framework-on-top-of-the-symfony2-components-part-1

Capítulo 1
Symfony2 es una serie de componentes reutilizables, desacoplados e independientes, escritos en PHP destinados a solucionar problemas comunes en el desarrollo web. En lugar de usar componentes de bajo nivel, puedes reutilizar los que propone el framework listos para usar o puedes crear tu propio framework.

¿Porqué querrías crear tu propio framework?

Muchos te dirían que es una locura, que tratas de reinventar la rueda, que en en vez de ello intenta reutilizar uno ya existente. La mayoría de las veces puede que tengan razón, pero he aquí un par de razones con el porqué crear un framework:
Aprender sobre la arquitectura de frameworks modernos para la web.
Refractorizar aplicaciones antiguas e incorporarles buenas prácticas del desarrollo web
Sencillamente te guiaré paso a paso cómo crear un framework web. Tendrás la ocasión de aprender un poco más sobre los componentes de symfony2. Muchos frameworks se llaman así mismos MVC. En este caso este trabajo está orientado simplemente hacia el mecanismo de los controladores. Pero lo deja abierto según como quieras incorporar el modelo, que puede ser con algún ORM o simplemente pdo, o la vista que la puedes manejar con algún motor de plantilla como por ejemplo Twig. Al crear un framework el hecho de perseguir si es MVC no debe ser una de las preocupaciones generales. La meta principal debe ser la separación de las preocupaciones, es lo que realmente creo de lo que los patrones de diseño deben realmente tener en cuenta. Los principios fundamentales de los componentes de Symfony2 están centrados en las especificaciones de HTTP. De esta manera los framework que estamos construyendo deben ser etiquetados como HTTP framework o simplemente frameworks de Petición/Respuesta.

Antes de comenzar
Para comenzar debemos tener instalado PHP en su versión más reciente, un servidor web,  un vasto conocimiento de PHP y sobre la programación orientada a objetos.

Bootstrapping
Primeramente definimos donde va a ser almacenado nuestro framework, cómo nombraremos nuestras clases y cómo referenciar dependencias externas. En este libro utilizaremos los estándares de código para Symfony2: Symfony2 Coding Standards

Instalación de componentes
La instalación de los componentes la realizaremos a través de Composer, así que los iremos listando en nuestro composer.json
{
“require”: {
“symfony/class-loader”: “2.1.*”
}
}

Luego de ello solo quedaría ejecutar el composer con:
$ php composer.phar install
Una vez ejecutado el comando aparecerá un nuevo directorio que debe contener el código del ClassLoader Component

Convenciones de nombre y autoloading:
Ahora vamos a autocargar todas las clases. Sin ello habría que hacer un requiere en cada archivo donde vayamos a utilizar determinada clase. Pero con algunas convenciones dejaremos que PHP haga el trabajo por nosotros. Symfony2 sigue el estándar PSR-0 para las convenciones de nombre y autocarga de clases. El componente ClassLoader de Symfony2 sigue este estándar en su mayoría.
Crea tu nuevo autoload.php
<?php

// framework/autoload.php

require_once __DIR__.’/vendor/symfony/class-loader/Symfony/Component/ClassLoader/UniversalClassLoader.php’;

use Symfony\Component\ClassLoader\UniversalClassLoader;

$loader = new UniversalClassLoader();
$loader->register();

Verifica que no tenga ningún error ejecutando php autoload.php. Es válido decir que composer crea un autoloader para todas las dependecias del proyecto enla carpeta vendor, por lo que sólo con require /vendor/composer/autoload.php.

Nuestro proyecto
En vez de crear un framework desde cero estaremos reescribiendo una y otra vez la misma aplicación.  Para empezar tenemos como ejemplo:
<?php

$input = $_GET['name'];

printf(‘Hello %s’, $_GET['name']);
En el próximo capítulo nos introduciremos en el componente HttpFoundation y veremos que tiene para nosotros.
