Crea tu propio framework, con lo último de los componentes de Symfony2.

Felix Ivan Romero Rodríguez Junio 30, 2014 Arquitectura y diseño, Patrones, Symfony 1 Comment

Aquí va la segunda entrega de la serie

De la serie original de Fabien en http://fabien.potencier.org/article/50/create-your-own-framework-on-top-of-the-symfony2-components-part-2

Capítulo 2

Antes de incursar en la refactorización de código, permíteme recordar porque es recomendable utilizar un framework y no una aplicación en puro PHP. Porque construir un framework a partir de componentes es mejor que construirlo desde cero. En la aplicación de ejemplo anterior existen ciertas irregularidades.
<?php

// framework/index.php

$input = $_GET['name'];

printf(‘Hello %s’, $input);

Primeramente si el parámetro no es servido en la url puede lanzar una advertencia (warning)
Se arreglaría de la siguiente forma:

$input = isset($_GET['name']) ? $_GET['name'] : ‘World’;

printf(‘Hello %s’, $input);

Sin embargo esto provoca que la aplicación no sea segura, ya que implica que la aplicación quede expuesta a ataques de tipo XSS. Una versión más segura es:
<?php

$input = isset($_GET['name']) ? $_GET['name'] : ‘World’;

header(‘Content-Type: text/html; charset=utf-8′);

printf(‘Hello %s’, htmlspecialchars($input, ENT_QUOTES, ‘UTF-8′));

Como debes conocer asegurando la aplicación con htmlspecialchars es tedioso y propenso a cometer errores. Ello es una de las razones que nos auxiliamos de un motor de plantillas como Twig. En Twig, el auto escape esta habilitado por defecto, esto puede ser una buena idea. Como podrás darte cuenta, el código inicial se ha vuelto complejo, a medida que hemos querido evitar errores y advertencias así como hacer menos vulnerable la aplicación. Más allá de la seguridad este código no se le pueden aplicar pruebas de manera fácil. Por pequeño que es el código, es complejo. Les muestro una tentativa de cómo sería escribir pruebas a este código.

<?php

// framework/test.php

class IndexTest extends \PHPUnit_Framework_TestCase
{
public function testHello()
{
$_GET['name'] = ‘Fabien’;

ob_start();
include ‘index.php’;
$content = ob_get_clean();

$this->assertEquals(‘Hello Fabien’, $content);
}
}

OOP con el componente HTTPFoundation
Escribiendo código Web se trata de la interacción con HTTP. De manera que los principios de la aplicación se centran sobre las especificaciones de HTTP. La especificación describe como un cliente interactúa con un servidor. El diálogo entre ambos se define por mensajes, peticiones y respuestas. En PHP una petición es representada por un conjunto de variables globales ($_GET, $_POST, $_FILE, $_COOKIE, $_SESSION…). Y la respuesta es generada por funciones (echo, header, setcookie, …). La principal función del componente HTTPFoundation es abstraer las variables globales y funciones en una capa orientada a objetos. Para utilizar este componente simplemente agregamos a nuestro composer.json:
{
“require”: {
“symfony/class-loader”: “2.1.*”,
“symfony/http-foundation”: “2.1.*”
}
}
Luego, ejecutamos la instalación:
php composer.phar update
Luego en el autoload se agrega el código necesario para cargar el componente
<?php

// framework/autoload.php

$loader->registerNamespace(‘Symfony\\Component\\HttpFoundation’, __DIR__.’/vendor/symfony/http-foundation’);

Ahora reescribimos la aplicación utilizando los objetos de Request y Response:
<php

// framework/index.php

require_once __DIR__.’/autoload.php’;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;

$request = Request::createFromGlobals();

$input = $request->get(‘name’, ‘World’);

$response = new Response(sprintf(‘Hello %s’, htmlspecialchars($input, ENT_QUOTES, ‘UTF-8′)));

$response->send();

El método createFromGlobals() crea el objeto Request a partir de las variables globales.
El método send() reenvía la respuesta hacia el cliente, imprimiendo primeramente las cabeceras. Antes de enviar la respuesta es necesario prepararla para asegurarse de que cumpla con las especificaciones de HTTP. Con la clase Request, se puede manipular todo lo referente a la petición y acceder a toda la información que posee:
<?php

// the URI being requested (e.g. /about) minus any query parameters
$request->getPathInfo();

// retrieve GET and POST variables respectively
$request->query->get(‘foo’);
$request->request->get(‘bar’, ‘default value if bar does not exist’);

// retrieve SERVER variables
$request->server->get(‘HTTP_HOST’);

// retrieves an instance of UploadedFile identified by foo
$request->files->get(‘foo’);

// retrieve a COOKIE value
$request->cookies->get(‘PHPSESSID’);

// retrieve an HTTP request header, with normalized, lowercase keys
$request->headers->get(‘host’);
$request->headers->get(‘content_type’);

$request->getMethod();    // GET, POST, PUT, DELETE, HEAD
$request->getLanguages(); // an array of languages the client accepts

You can also simulate a request:
$request = Request::create(‘/index.php?name=Fabien’);

Con la clase Response puedes controlar la respuesta que es devuelta:
<?php

$response = new Response();

$response->setContent(‘Hello world!’);
$response->setStatusCode(200);
$response->headers->set(‘Content-Type’, ‘text/html’);

// configure the HTTP cache headers
$response->setMaxAge(10);
Para depurar la Respuesta se puede representar como una cadena de texto, mostrará las cabeceras y el contenido.
Por último pero no menos importante, las clases han de ser auditadas para comprobar la seguridad. Algo tan sencillo como obtener una dirección IP puede ser inseguro.
<?php

if ($myIp == $_SERVER['REMOTE_ADDR']) {
// the client is a known one, so give it some more privilege
}

Utilizando la función Request::getClientIp() , se da el comportamiento correcto para solicitar esta información, incluso si en algún momento se cambian los proxies.

Este método trabaja de manera segura según las circunstancias. Puedes utilizarlo en tus proyectos que sin importar la configuración se ejecutará de forma segura. Ello es una de las ventajas de utilizar un framework. Si construyeras todo desde cero, tendrías que ponerte a diseñar como enfrentarte a este problema, es mejor utilizar la tecnología que te rodea. Si quieres saber más de este componente puedes dirigirte a la documentación donde se explica detalladamente.
