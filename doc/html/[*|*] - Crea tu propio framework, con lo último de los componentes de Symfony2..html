Crea tu propio framework, con lo último de los componentes de Symfony2.

Felix Ivan Romero Rodríguez Mayo 20, 2015 Comunidades, Desarrollo web, PHP, Symfony No comments

Inyección de dependencias…una herramienta que en Symfony2 es explotada intensamente. En esta última entrega veremos como incorporar un contenedor de inyección de dependencias en nuestro framework y las declaraciones de dependencias. Espero que hayan aprendido y se embullen a construir su propio framework. Sin más:

Capítulo 12
En el capítulo anterior vaciamos la clase framework y la pusimos a heredar de httkernel, así que viéndola vacía, quizás te motive a utilizarla un poco

<?php

// example.com/src/Simplex/Framework.php

namespace Simplex;

use Symfony\Component\Routing;
use Symfony\Component\HttpKernel;
use Symfony\Component\EventDispatcher\EventDispatcher;

class Framework extends HttpKernel\HttpKernel
{
public function __construct($routes)
{
$context = new Routing\RequestContext();
$matcher = new Routing\Matcher\UrlMatcher($routes, $context);
$resolver = new HttpKernel\Controller\ControllerResolver();

$dispatcher = new EventDispatcher();
$dispatcher->addSubscriber(new HttpKernel\EventListener\RouterListener($matcher));
$dispatcher->addSubscriber(new HttpKernel\EventListener\ResponseListener('UTF-8'));

parent::__construct($dispatcher, $resolver);
}
}

Logrando que el controlador frontal  se comporten de manera más concisa al quedar:


<?php

// example.com/web/front.php

require_once __DIR__.'/../vendor/.composer/autoload.php';

use Symfony\Component\HttpFoundation\Request;

$request = Request::createFromGlobals();
$routes = include __DIR__.'/../src/app.php';

$framework = new Simplex\Framework($routes);

$framework->handle($request)->send();

Ello permite que se pueda concebir más de una aplicación bajo el mismo framework. ¿Utilidad?. Permite también establecer el mecanismo de los entornos de desarrollo y de producción. En entorno es conveniente depurar el funcionamiento de la aplicación  y en producción evitar este tipo de configuración. Muy bien, pero mover el código hacia nuestra clase, permite incrementar las opciones de configuración, pero…aleja la accesibilidad a elementos como el despachador de eventos, haciendo que pierda flexibilidad y entonces habría que realizar una cadena de cambios horizontales en nuestras clases para que la flexibilidad no se vea en reflujo…y entonces…bueno, ahora es cuando usamos la inyección de dependencia, donde declaramos lo que vayamos a usar  y usémosle…donde iniciamos..pues en el composer:

{
"require": {
"symfony/class-loader": "2.1.*",
"symfony/http-foundation": "2.1.*",
"symfony/routing": "2.1.*",
"symfony/http-kernel": "2.1.*",
"symfony/event-dispatcher": "2.1.*",
"symfony/dependency-injection": "2.1.*"
},
"autoload": {
"psr-0": { "Simplex": "src/", "Calendar": "src/" }
}
}

Luego de ello crear nuestro contenedor de dependencias:

<?php

// example.com/src/container.php

use Symfony\Component\DependencyInjection;
use Symfony\Component\DependencyInjection\Reference;

$sc = new DependencyInjection\ContainerBuilder();
$sc->register('context', 'Symfony\Component\Routing\RequestContext');
$sc->register('matcher', 'Symfony\Component\Routing\Matcher\UrlMatcher')
->setArguments(array($routes, new Reference('context')))
;
$sc->register('resolver', 'Symfony\Component\HttpKernel\Controller\ControllerResolver');

$sc->register('listener.router', 'Symfony\Component\HttpKernel\EventListener\RouterListener')
->setArguments(array(new Reference('matcher')))
;
$sc->register('listener.response', 'Symfony\Component\HttpKernel\EventListener\ResponseListener')
->setArguments(array('UTF-8'))
;
$sc->register('listener.exception', 'Symfony\Component\HttpKernel\EventListener\ExceptionListener')
->setArguments(array('Calendar\\Controller\\ErrorController::exceptionAction'))
;
$sc->register('dispatcher', 'Symfony\Component\EventDispatcher\EventDispatcher')
->addMethodCall('addSubscriber', array(new Reference('listener.router')))
->addMethodCall('addSubscriber', array(new Reference('listener.response')))
->addMethodCall('addSubscriber', array(new Reference('listener.exception')))
;
$sc->register('framework', 'Simplex\Framework')
->setArguments(array(new Reference('dispatcher'), new Reference('resolver')))
;

return $sc;

La meta..configurar todas nuestras dependencias. Por ejemplo para instanciar un objeto de tipo RouterListeners hace falta que tenga un objeto de tipo matcher (new Reference(‘matcher’). Y ahora nuestro controlador frontal, se encarga de crear las conexiones

<?php

// example.com/web/front.php

require_once __DIR__.'/../vendor/.composer/autoload.php';

use Symfony\Component\HttpFoundation\Request;

$routes = include __DIR__.'/../src/app.php';
$sc = include __DIR__.'/../src/container.php';

$request = Request::createFromGlobals();

$response = $sc->get('framework')->handle($request);

$response->send();

Y la clase framework retorna a su version original, solo heredando.
Apenas hemos arañado la superficie de todo lo que se puede realizar con la inyección de dependencias, este componente permite trabajar con cualquier tipo de clase escrita en PHP. De esta manera concluye nuestra travesía en la construción de un framework PHP
