Crea tu propio framework, con lo último de los componentes de Symfony2.

Felix Ivan Romero Rodríguez Agosto 26, 2014 Patrones, PHP, Symfony No comments

Llegando de vacaciones, les traigo el capítulo 3 de la serie:

De la serie original de Fabien en http://fabien.potencier.org/article/50/create-your-own-framework-on-top-of-the-symfony2-components-part-3

Hasta ahora nuestra aplicación es bastante simple, conteniendo una sola página. Para condimentar un poco las cosas adicionemos una página de despedida.

<?php

// framework/bye.php

require_once __DIR__.’/autoload.php’;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;

$request = Request::createFromGlobals();

$response = new Response(‘Goodbye!’);
$response->send();

Como se aprecia, mucho de este código es similar al que tiene la primera página. Vamos a extraer el código que es común para todas páginas. Haciendo esto es un buen plan para acercarnos un poco más a un framework real. La manera de extraerlo es precisamente a través de archivos de inclusión.

<?php

// framework/init.php

require_once __DIR__.’/autoload.php’;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;

$request = Request::createFromGlobals();
$response = new Response();

Y ahora lo vemos en acción:

<?php

// framework/index.php

require_once __DIR__.’/init.php’;

$input = $request->get(‘name’, ‘World’);

$response->setContent(sprintf(‘Hello %s’, htmlspecialchars($input, ENT_QUOTES, ‘UTF-8′)));
$response->send();

Y la página de despedida diría lo siguiente:

<?php

// framework/bye.php

require_once __DIR__.’/init.php’;

$response->setContent(‘Goodbye!’);
$response->send();

Ya hemos separado el código común en un punto global, sin embargo, aún no se lleva la idea de una abstracción completa, puesto que por ejemplo aún usamos el método send() en las páginas, estas no se ven como plantillas y aún no son capaces de ser probadas correctamente. Por otra parte adicionar otra página implica crear otro script PHP, el cual estaría expuesto vía la URL (http://example.com/bye.php) Existe una correspondencia directa entre el script PHP y la URL provista por el usuario. Ello sucede ya que el servidor web es quien se encarga de despachar directamente la petición realizada. Sería una buena idea mover todo el proceso de despacho de la petición hacia nuestro código, para que este gane en flexibilidad. Ello se logra enrutando todas las peticiones hacia un único archivo PHP. De esta manera sólo se expone al usuario un único archivo y esta manera de procesar las peticiones se le conoce como el patrón de diseño “controlador frontal”. Así que el código se vería de la siguiente forma:

<?php

// framework/front.php

require_once __DIR__.’/autoload.php’;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;

$request = Request::createFromGlobals();
$response = new Response();

$map = array(
‘/hello’ => __DIR__.’/hello.php’,
‘/bye’   => __DIR__.’/bye.php’,
);

$path = $request->getPathInfo();
if (isset($map[$path])) {
require $map[$path];
} else {
$response->setStatusCode(404);
$response->setContent(‘Not Found’);
}

$response->send();

Y aquí un ejemplo de como quedaría hello.php:

<?php

// framework/hello.php

$input = $request->get(‘name’, ‘World’);
$response->setContent(sprintf(‘Hello %s’, htmlspecialchars($input, ENT_QUOTES, ‘UTF-8′)));
En el script que se llama front.php, $map asocia las rutas de la URL con su correspondiente script PHP. Como valor agregado si las rutas no existes se lanzan errores 404. Ahora tenemos el control del sitio.
Para acceder a una página, se debe utilizar el front.php:

http://example.com/front.php/hello?name=Fabien

http://example.com/front.php/bye

/hello y /bye son ahora rutas.
Servidores web, como apache o nginx pueden reescribir las direcciones de entrada, de manera que remueven el nombre del controlador por lo que se puede escribir http://example.com/hello?name=Fabien lo cual es mucho mejor.
El truco está en el uso de Request::getPathInfo(), método que retorna la ruta de la petición sin el nombre del script del controlador.
Ni siquiera es necesario configurar el servidor web para probar el código, en lugar de ello se puede reemplazar: request = Request::createFromGlobals();  por $request = Request::create(‘/hello?name=Fabien’);
Ahora el servidor web accede al mismo script PHP, por lo que podemos quitar los script restantes de la carpeta pública (web) y configurar nuestro servidor web que apunte  hacia la carpeta pública y el cliente no acceda a otra dirección:
example.com
??? composer.json
?   src
?   ??? autoload.php
?   ??? pages
?       ??? hello.php
?       ??? bye.php
??? vendor
??? web
??? front.php

Lo otro que se repite en cada página es el método setContent, para convertir nuestras páginas en plantillas sólo hay que mostrarlas con echo directamente desde el controlador frontal

<?php

// example.com/web/front.php

// …

$path = $request->getPathInfo();
if (isset($map[$path])) {
ob_start();
include $map[$path];
$response->setContent(ob_get_clean());
} else {
$response->setStatusCode(404);
$response->setContent(‘Not Found’);
}

// …
Y el script hello puede ser convertido en plantilla:
<!– example.com/src/pages/hello.php –>

<?php $name = $request->get(‘name’, ‘World’) ?>

Hello <?php echo htmlspecialchars($name, ENT_QUOTES, ‘UTF-8′) ?>

Y ya tenemos nuestro framework de hoy
<?php

// example.com/web/front.php

require_once __DIR__.’/../src/autoload.php’;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;

$request = Request::createFromGlobals();
$response = new Response();

$map = array(
‘/hello’ => __DIR__.’/../src/pages/hello.php’,
‘/bye’   => __DIR__.’/../src/pages/bye.php’,
);

$path = $request->getPathInfo();
if (isset($map[$path])) {
ob_start();
include $map[$path];
$response->setContent(ob_get_clean());
} else {
$response->setStatusCode(404);
$response->setContent(‘Not Found’);
}

$response->send();

Adicionar una nueva página consta de dos pasos: agregar una entrada en el $map y crear una plantilla PHP. Desde la plantilla se pueden obtener los datos de la Request a través de la variable $request y cambiar el Response desde la variable Response. Si decides detenerte aqui, mejoraría el framework llevando el $map hacia un archivo de configuración.
